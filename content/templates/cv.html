CV

   2005 - 2011    Computer Engineering student at Universitat Jaume I (UJI)

I studied Computer Engineering in the Universitat Jaume I (UJI) in Castellón, Spain. Focused on software engineering and algorithms.

   2011 - 2014    Software Engineer at TUENTI TECHNOLOGIES    Madrid (Spain)

One of the first projects I work on at the company was a new sign up system with phone verification for the social network. The project was built using a SMS gateway with security code generation to verify the identity of the profile.

Another web project I worked on was the rewrite of the social events feature. Rebuilding both, frontend and backend, focusing mainly in its scalability to be able to manage the huge load of the site: database denormalization, model caching, load balancing, data archiving, etc.

Then, I spent a full year working on the social network chat. There were two main projects our team developed: The group chat, extending the existing one-to-one chat to allow users to create roms with more than two members. And the photo sharing, with which the users could send their photos through a chat conversation.

Also worked for a year and a half on the MVNO (Mobile Virtual Network Operator) side of the company. Developing with the rest of the acquisition team an A/B testing framework for our online store checkout with payment integration, identity verification and shipment tracking.

At the MVNO, our team also rebuilt the retail sales tool to solve security issues, and perform some usability improvements from the feedback provided by the sellers. Also improving and standarizing the backend API the tool was relying on.

All the work on the server side was done in PHP, using an in house web framework and the Symfony 2 framework. On the storage side, MySQL, Memcached and HBase were the main tools. We were also relying on Beanstalkd as messaging system and queue processing. On the client side, we developed in JavaScript with both jQuery and YUI.

The team were managed with Scrum as long as custom methodologies. We did heavy use of continuous integration and automated development environments based on Vagrant that mimicked production. High quality testing of every project became a self imposed standard.

   2015 - 2017    Software Engineer at QDQ Media    Madrid (Spain)

At the beginning of joining the company, I developed a sign in solution to bring customers from other companies in partnership. It was developed in Python 3 and Django framework.

Next my team analyzed the architecture of the company systems, and decided to start the migration process to a microservices architecture. We wanted to start small, and I built the asynchronous email service fed up through RabbitMQ queues. That also implied to refactor the monolithic architecture to adapt to an asynchronous pattern when using the service. The stack used was Python 3 with Django, Postgres and RabbitMQ.

Continuing with the microservices approach, I worked on a substitution service for an old call tracking system. This time, building a synchronous system with an authenticated HTTP API, so multiple user facing products could benefit from the same service easily. The tech here was Node.js with the hapi web framework.

Another big project that I worked on was a full revamp of the in house statistics service. Starting with the frontend standarization of the tracking across different projects, adapting their templates and building an agnostic client-side to automatically start tracking when included. The legacy system was using redis to store the data, and we migrated it to Riak KV.

To support this kind of infrastructure, we use a private PaaS (Platform-as-a-Service) based on docker (like our development environments) to quickly release new applications with minimal operations effort. We also established an automated release process with continuous integration.

   2017    Software Engineer at GIGAS Hosting    Madrid (Spain)

My role at GIGAS was to analyze the status of the projects modernize some tools and processes.

I kickstarted the testing of the Python codebase with Pytest and developed some plugins to fit the various project needs. Also, designed the continuous integration procedure based on Git Flow to speed up integration times.

For the renovation of the tools, provisioned a GitLab service to improve code reviews, continuous testing, continuous integration, issue tracking,...

The most insightful side of my work was to train the existing team in the new techniques and procedures. To transmit them the concept of quality and testing, and the confidence that arises from sticking to good practices.

   2018    Site Reliability Engineer at Telefónica Digital    Madrid (Spain)     Now

Moving to a more systems oriented career, I rejoined my old team at TUENTI TECHNOLOGIES, acquired by Telefónica.
